更新时间：2026-05-26

## 当前目标

本项目是信息检索课程最终大作业的协作骨架，目标是实现一个面向苏州大学导师信息的垂直检索系统。当前阶段重点不是追求完整数据和高级算法，而是先搭建可运行、可分工、可逐步扩展的通用架构，方便 4 名组员在 GitHub 上并行开发。

## 当前已实现内容

### 1. 项目基础结构

已经建立 Python 项目结构：

- `suda_ir/crawler/`：爬虫和 HTML 解析模块。
- `suda_ir/data/`：教师数据读写模块。
- `suda_ir/ir/`：分词、索引、BM25 检索和搜索接口。
- `suda_ir/app/`：命令行入口和可选 Streamlit Web 界面。
- `data/sample/`：示例数据。
- `tests/`：基础单元测试。
- `docs/`：项目分工、实验记录和进度文档。

### 2. 数据模型

已经定义统一的教师文档模型 `TeacherDoc`，主要字段包括：

- `doc_id`：文档 ID。
- `name`：教师姓名。
- `college`：学院。
- `title`：职称。
- `research`：研究方向。
- `papers`：论文或成果。
- `profile`：个人简介。
- `content`：清洗后的全文正文。
- `url` / `final_url`：原始 URL 和跳转后的最终 URL。
- `extra`：其他扩展字段。

这个设计支持“字段能抽取就结构化，抽取失败就用全文兜底”的策略，适合处理学校官网中格式不统一的教师页面。

### 3. 基础爬虫与解析

当前已提供基础实现：

- 支持从种子 URL 出发递归抓取静态页面。
- 支持 HTTP 跳转后的最终 URL 记录。
- 支持同域名链接发现。
- 支持仅抓取 `.htm`、`.html` 或目录型静态页面。
- 支持保存原始 HTML 到 `data/raw/`。
- 支持将解析后的教师文档保存为 JSONL。
- 支持正文提取和简单字段抽取。
- 支持电话和邮箱脱敏，将敏感信息替换为 `***`。

当前解析规则是基础版本，只能处理一部分常见字段格式。后续需要根据真实学院页面继续补充规则。

### 4. 基础检索系统

当前已实现基础 BM25 检索：

- 支持中文分词，优先使用 `jieba`。
- 如果未安装 `jieba`，会使用简单字符 n-gram 作为兜底。
- 支持字段加权：
  - 姓名权重最高；
  - 研究方向权重较高；
  - 论文、职称、简介、学院、全文参与综合排序。
- 支持姓名精确检索。
- 支持学院字段过滤。
- 支持综合查询，例如研究方向、关键词、论文相关词等。

### 5. 运行入口

当前有两个入口：

命令行检索：

```bash
python -m suda_ir.app.cli --data data/sample/teachers.jsonl --query 自然语言处理
python -m suda_ir.app.cli --data data/sample/teachers.jsonl --query 周国栋 --field name
```

可选 Web 界面：

```bash
streamlit run suda_ir/app/streamlit_app.py
```

### 6. 测试

当前已有基础测试：

- 姓名精确检索。
- 研究方向检索。
- 学院字段过滤。

已验证测试通过：

```bash
python -m unittest discover -s tests
```

## 重要说明：当前数据是模拟数据

当前 `data/sample/teachers.jsonl` 中的数据是为了验证系统流程而手工构造的模拟数据，不是从苏州大学官网真实爬取的数据。

这些示例数据只用于：

- 检查项目能否运行。
- 验证数据模型是否可用。
- 验证 BM25 检索流程是否正常。
- 给组员提供输入数据格式示例。

后续正式实验必须替换为从苏州大学官网爬取、清洗、脱敏后的真实教师数据。真实数据建议保存到：

```text
data/processed/teachers.jsonl
```

该目录默认不提交到 GitHub，避免误传大规模网页数据或隐私信息。
